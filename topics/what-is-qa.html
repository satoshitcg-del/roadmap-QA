QA คืออะไร?

QA (Quality Assurance) คือกระบวนการรับประกันคุณภาพซอฟต์แวร์ โดยมุ่งเน้นการป้องกันข้อบกพร่องตั้งแต่ต้น ผ่านกระบวนการและมาตรฐานที่ดี มากกว่าการหาบั๊กหลังจากพัฒนาเสร็จแล้ว

💡
Key Concept: QA มุ่งเน้น "ป้องกัน" ข้อผิดพลาด ส่วน QC มุ่งเน้น "ตรวจจับ" ข้อผิดพลาด

QA vs QC: ความแตกต่าง

ด้าน QA (Quality Assurance) QC (Quality Control)
จุดเน้น กระบวนการ (Process) ผลิตภัณฑ์ (Product)
เป้าหมาย ป้องกันข้อผิดพลาด หาและแก้ไขข้อผิดพลาด
เวลา ตลอด SDLC หลังจากพัฒนา
กิจกรรม Process definition, Audits Testing, Reviews

ทักษะที่ QA Engineer ต้องมี

💻

Technical Skills

  • เข้าใจ Testing Methodologies
  • เขียน Test Cases
  • ใช้ Automation Tools (Selenium, etc.)
  • รู้จัก API และ Database
  • เขียน SQL
  • Version Control (Git)
👤

Soft Skills

  • Analytical Thinking
  • Attention to Detail
  • Communication Skills
  • Problem Solving
  • Time Management
  • Teamwork

Career Path

1

Junior QA (0-2 ปี)

Execute test cases, Report bugs, Learn tools

↓
2

QA Engineer (2-5 ปี)

Write test plans, Automation, API testing

↓
3

Senior QA (5+ ปี)

Test strategy, Mentoring, CI/CD integration

↓
4

QA Lead / Manager

Team management, Process improvement, Strategy
