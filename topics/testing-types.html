🧩 Testing Types

เรียนรู้ประเภทการทดสอบต่างๆ: Functional, Non-Functional, Manual, Automation และอื่นๆ

📋 สารบัญ

  • Functional Testing
  • Non-Functional Testing
  • Manual vs Automation
  • Levels of Testing
  • แบบทดสอบ

Functional Testing

การทดสอบฟังก์ชันการทำงานของซอฟต์แวร์ว่าตรงตาม Requirements หรือไม่

📦

Unit Testing

ทดสอบส่วนย่อยที่สุด (function, method) มักโดย Developer

🔗

Integration Testing

ทดสอบการเชื่อมต่อระหว่าง modules/components

🎯

System Testing

ทดสอบระบบทั้งหมดตาม Requirements

✅

Acceptance Testing

ตรวจสอบว่าพร้อมส่งมอบ (UAT, BAT)

Non-Functional Testing

ทดสอบคุณสมบัติที่ไม่ใช่ฟังก์ชันการทำงานโดยตรง เช่น ความเร็ว ความปลอดภัย

ประเภท จุดประสงค์ เครื่องมือตัวอย่าง
Performance ความเร็ว ความเสถียร JMeter, LoadRunner
Security ความปลอดภัย OWASP ZAP, Burp Suite
Usability ความง่ายต่อการใช้งาน User testing
Compatibility ทำงานบนอุปกรณ์/บราว์เซอร์ต่างๆ BrowserStack
Reliability ความน่าเชื่อถือ Chaos Engineering

Manual vs Automation Testing

✋

Manual Testing

  • ✓ Exploratory testing
  • ✓ Usability testing
  • ✓ Ad-hoc testing
  • ✗ Time-consuming
  • ✗ Prone to human error
🤖

Automation Testing

  • ✓ Regression testing
  • ✓ Load testing
  • ✓ Repetitive tasks
  • ✓ Fast execution
  • ✗ Initial setup cost
💡
แนะนำ: ใช้ทั้งสองแบบร่วมกัน — Manual สำหรับการทดสอบที่ต้องใช้มนุษย์ตัดสิน และ Automation สำหรับงานที่ทำซ้ำบ่อย

Levels of Testing (Testing Pyramid)

        /\
       /  \       UI Tests (E2E)
      / E2E \      - Few tests
     /________\     - Slow, expensive
    /          \
   / Integration \   Integration Tests
  /________________\  - Service/API level
 /                  \
/     Unit Tests     \ Unit Tests
______________________ - Many tests
                       - Fast, cheap

The Testing Pyramid: มี Unit Tests มากที่สุด (ฐานแพร่) และ E2E Tests น้อยที่สุด (ยอดแหลม)
