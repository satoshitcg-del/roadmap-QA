Equivalence Partitioning (EP)

แบ่งข้อมูลออกเป็นกลุ่มที่เหมือนกัน (equivalent) แล้วเลือกตัวแทนมาทดสอบจากแต่ละกลุ่ม

ตัวอย่าง: ช่องรับอายุ 1-100 ปี
  • Valid EP: 1-100
  • Invalid EP: <1, >100, non-numeric

Boundary Value Analysis (BVA)

ทดสอบที่ค่าขอบเขต (boundaries) เพราะมักพบ defects ที่นี่

ตัวอย่าง: ช่วง 1-100
  • Test values: 0, 1, 2, 99, 100, 101

Decision Table

สร้างตารางแสดง conditions และ actions ทั้งหมด ช่วยให้ไม่พลาด combination ใดๆ

State Transition

ทดสอบการเปลี่ยน state ของระบบ เช่น Order: New → Paid → Shipped → Delivered
