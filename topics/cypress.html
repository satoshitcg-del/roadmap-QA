🌲 Cypress คืออะไร?

Cypress เป็น JavaScript End-to-End Testing Framework ที่ทันสมัย ออกแบบมาเฉพาะสำหรับ Modern Web Applications

🆚 Cypress vs Selenium

CypressSelenium
JavaScript เท่านั้นหลายภาษา (Python, Java, etc.)
Auto-wait (รออัตโนมัติ)ต้องเขียน Explicit Wait เอง
Real-time reloadต้องรันใหม่ทุกครั้ง
Debug ง่าย (Time Travel)Debug ธรรมดา
รองรับ Chrome, Edge, Firefoxรองรับทุก Browser
ไม่รองรับ Mobileรองรับ Mobile ผ่าน Appium
เร็วกว่าช้ากว่า (network latency)

✨ จุดเด่นของ Cypress

  • ⚡ Fast - เร็วกว่า Selenium มาก (รันใน browser เดียวกัน)
  • 🔍 Time Travel Debugging - ย้อนกลับดู state ทีละ step
  • 📸 Automatic Screenshots/Videos - ถ่ายรูป/วิดีโออัตโนมัติตอน fail
  • ⏱️ Auto-wait - รอ element โดยอัตโนมัติ
  • 🎯 Real-time Reloads - แก้โค้ดแล้วรันใหม่ทันที

⚙️ การติดตั้ง Cypress

# 1. สร้างโปรเจกต์ใหม่
mkdir my-cypress-project
cd my-cypress-project
npm init -y

# 2. ติดตั้ง Cypress
npm install cypress --save-dev

# 3. เปิด Cypress Test Runner
npx cypress open

# หรือ รันผ่าน command line (headless)
npx cypress run

📁 โครงสร้างโปรเจกต์ Cypress

cypress/
├── e2e/              # Test files (End-to-End tests)
│   ├── login.cy.js
│   └── checkout.cy.js
├── fixtures/         # Test data (JSON files)
│   └── users.json
├── support/

cypress.config.js     # Cypress configuration

describe('Login Tests', () => {
  
  // รันทุก test ก่อน
  beforeEach(() => {
    cy.visit('/login')
  })

  it('should login with valid credentials', () => {
    // Test steps ที่นี่
  })

  it('should show error with invalid credentials', () => {
    // Test steps ที่นี่
  })
})

🚀 Hello World - ทดสอบ Login

describe('Login Page', () => {
  
  beforeEach(() => {
    cy.visit('https://example.com/login')
  })

  it('successfully logs in with valid credentials', () => {
    // หาและกรอก username
    cy.get('[data-testid="username-input"]')
      .type('testuser')
    
    // กรอก password
    cy.get('[data-testid="password-input"]')
      .type('Test@123')
    
    // คลิก login
    cy.get('[data-testid="login-button"]')
      .click()
    
    // ตรวจสอบว่า redirect ไป dashboard
    cy.url()
      .should('include', '/dashboard')
    
    // ตรวจสอบว่าแสดง welcome message
    cy.contains('Welcome, testuser!')
      .should('be.visible')
  })
})

🎯 Selectors และ Commands

การหา Elements (cy.get, cy.find, cy.contains)

// หาด้วย CSS Selector
cy.get('.button-primary')
cy.get('#submit-btn')
cy.get('[data-testid="login-form"]')  // ✅ แนะนำ

// หาด้วยข้อความ
cy.contains('Submit')
cy.contains('button', 'Submit')  // หา button ที่มีข้อความ Submit

// Chain commands
cy.get('.form')
  .find('input[name="email"]')  // หาภายใน parent
  .type('test@example.com')

// การกรอง
cy.get('li')
  .first()      // อันแรก
  .last()       // อันสุดท้าย
  .eq(2)        // index ที่ 2
  .filter('.active')  // ที่มี class active
  .not('.disabled')   // ที่ไม่มี class disabled

🎮 Common Commands

Commandคำอธิบายตัวอย่าง
.visit()เปิดหน้าเว็บcy.visit('/login')
.click()คลิก elementcy.get('button').click()
.type()พิมพ์ข้อความcy.get('input').type('hello')
.clear()ล้างข้อความcy.get('input').clear()
.select()เลือก dropdowncy.get('select').select('Option 1')
.check()/uncheck()เลือก/ยกเลิก checkboxcy.get('[type="checkbox"]').check()
wait()รอ (ไม่แนะนำ ใช้ auto-wait)cy.wait(1000)

⌨️ Special Keys ใน .type()

cy.get('input').type('Hello{enter}')      // พิมพ์แล้วกด Enter
cy.get('input').type('{backspace}')         // กด Backspace
cy.get('input').type('{esc}')               // กด Escape
cy.get('input').type('{ctrl+c}')            // Ctrl+C
cy.get('input').type('{selectall}{del}')    // เลือกทั้งหมดแล้วลบ

✅ Assertions (Implicit & Explicit)

Cypress มี Assertions ในตัว ไม่ต้อง import library เพิ่ม

Implicit Assertions (.should())

// Visibility
cy.get('.message').should('be.visible')
cy.get('.loading').should('not.be.visible')

// Existence
cy.get('.user-profile').should('exist')
cy.get('.error').should('not.exist')

// State
cy.get('button').should('be.enabled')
cy.get('input').should('be.disabled')
cy.get(':checkbox').should('be.checked')

// Content
cy.get('.title').should('have.text', 'Welcome')
cy.get('.title').should('contain', 'Welcome')  // บางส่วนก็ได้
cy.get('input').should('have.value', 'test@email.com')

// Class/CSS
cy.get('.button').should('have.class', 'primary')
cy.get('.banner').should('have.css', 'background-color', 'rgb(255, 0, 0)')

// Attribute
cy.get('a').should('have.attr', 'href', '/dashboard')
cy.get('img').should('have.attr', 'alt')

// Length
cy.get('.list-item').should('have.length', 3)
cy.get('.list-item').should('have.length.at.least', 1)

// Chain multiple assertions
cy.get('.user-card')
  .should('be.visible')
  .and('have.class', 'active')
  .and('contain', 'John Doe')

Explicit Assertions (expect, assert)

// BDD Style (expect)
cy.get('.user').then(($user) => {
  expect($user).to.have.text('John Doe')
  expect($user).to.be.visible
})

// ตรวจสอบ API Response
cy.request('GET', '/api/users/1').then((response) => {
  expect(response.status).to.eq(200)
  expect(response.body).to.have.property('id', 1)
  expect(response.body.name).to.be.a('string')
  expect(response.body.email).to.include('@')
})

🚀 Advanced Features

📡 API Testing ใน Cypress

// Intercept/Stub API calls
cy.intercept('GET', '/api/users', {
  statusCode: 200,
  body: { users: [{ id: 1, name: 'John' }] }
}).as('getUsers')

cy.visit('/users')
cy.wait('@getUsers')  // รอ API เรียกเสร็จ

// ตรวจสอบ API ถูกเรียก
cy.get('@getUsers').its('request.url').should('include', '/api/users')

🔧 Custom Commands

// cypress/support/commands.js
Cypress.Commands.add('login', (email, password) => {
  cy.visit('/login')
  cy.get('[data-testid="email"]').type(email)
  cy.get('[data-testid="password"]').type(password)
  cy.get('[data-testid="login-btn"]').click()
  cy.url().should('include', '/dashboard')
})

// ใช้ใน test
cy.login('test@example.com', 'password123')

📁 Fixtures (Test Data)

// cypress/fixtures/users.json
{
  "admin": {
    "email": "admin@test.com",
    "password": "admin123"
  },
  "user": {
    "email": "user@test.com",
    "password": "user123"
  }
}

// ใช้ใน test
cy.fixture('users').then((users) => {
  cy.login(users.admin.email, users.admin.password)
})
