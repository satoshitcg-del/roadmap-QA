👤 Usability Testing คืออะไร?

Usability Testing คือการทดสอบว่าผู้ใช้สามารถใช้งานระบบได้ง่าย มีประสิทธิภาพ และพึงพอใจหรือไม่ โดยการให้ผู้ใช้จริงลองใช้งานและสังเกตพฤติกรรม

🎯 5 องค์ประกอบของ Usability (ISO 9241)

  1. Learnability - เรียนรู้ใช้งานได้ง่าย
  2. Efficiency - ทำงานได้รวดเร็ว
  3. Memorability - จำวิธีใช้ได้หลังไม่ได้ใช้นาน
  4. Errors - ทำผิดพลาดน้อย และแก้ไขง่าย
  5. Satisfaction - พึงพอใจในการใช้งาน

🧪 Usability Testing Methods

👁️

Moderated Testing

มี moderator คอยสังเกตและถามคำถาม

💻

Unmoderated Testing

ผู้ใช้ทดสอบเองผ่านเครื่องมือ เช่น Maze, UserTesting

🏠

Remote Testing

ทดสอบผ่านออนไลน์ ไม่ต้องมาที่เดียวกัน

🏢

In-Person Testing

ทดสอบแบบเจอตัว สังเกต body language

📝 Test Tasks ตัวอย่าง

1. "คุณต้องการซื้อเสื้อผ้า ช่วยหาเสื้อเชิ้ตสีขาว size M และใส่ตะกร้า"

2. "คุณลืมรหัสผ่าน ช่วย reset password"

3. "คุณต้องการยกเลิกคำสั่งซื้อ #12345"

4. "หาข้อมูลการจัดส่งสินค้าของคำสั่งซื้อล่าสุด"

📋 Nielsen's 10 Usability Heuristics

  1. Visibility of System Status - แสดงสถานะให้เห็น (loading, progress)
  2. Match Between System and Real World - ใช้ภาษาที่ผู้ใช้เข้าใจ
  3. User Control and Freedom - มี Undo/Exit/Back ได้
  4. Consistency and Standards - ใช้ conventions มาตรฐาน
  5. Error Prevention - ป้องกันข้อผิดพลาดก่อนเกิด
  6. Recognition Rather Than Recall - เห็นตัวเลือก ไม่ต้องจำ
  7. Flexibility and Efficiency of Use - มี shortcuts สำหรับผู้เชี่ยวชาญ
  8. Aesthetic and Minimalist Design - ไม่มีข้อมูลเกินจำเป็น
  9. Help Users Recognize and Recover from Errors - Error messages ชัดเจน
  10. Help and Documentation - มีคู่มือช่วยเหลือที่หาได้ง่าย

📊 Usability Metrics

Metric คำอธิบาย วิธีวัด
Task Success Rate % ของผู้ใช้ที่ทำงานสำเร็จ (สำเร็จ / ทั้งหมด) × 100
Time on Task เวลาที่ใช้ทำแต่ละงาน Stopwatch / Tool
Error Rate จำนวนข้อผิดพลาดต่งาน นับ slip errors, mistakes
SUS Score System Usability Scale (0-100) แบบสอบถาม 10 ข้อ
NPS Net Promoter Score "แนะนำเพื่อนไหม?" -10 ถึง 10

📝 SUS (System Usability Scale) Questions

  1. I think that I would like to use this system frequently
  2. I found the system unnecessarily complex
  3. I thought the system was easy to use
  4. I think that I would need the support of a technical person
  5. I found the various functions in this system were well integrated
  6. I thought there was too much inconsistency in this system
  7. I would imagine that most people would learn to use this system very quickly
  8. I found the system very cumbersome to use
  9. I felt very confident using the system
  10. I needed to learn a lot of things before I could get going

Scoring: ถ้าได้ > 68 ถือว่า Above Average
