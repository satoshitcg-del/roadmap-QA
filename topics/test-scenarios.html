Test Scenario vs Test Case

Test Scenario Test Case
High-level description Detailed step-by-step
"What to test" "How to test"
One scenario = many cases One case = specific test

การเขียน Test Scenario

Test Scenario ควรครอบคลุมความต้องการทั้งหมด โดยไม่ซ้ำซ้อน

ตัวอย่าง Test Scenarios สำหรับ Login 
TS1: Login ด้วย credentials ที่ถูกต้อง
TS2: Login ด้วย password ผิด
TS3: Login ด้วย username ที่ไม่มีในระบบ
TS4: Login เมื่อระบบ offline
TS5: Forgot Password functionality

การเขียน Test Case

Test Case ที่ดีต้องมี:

  • Test Case ID - รหัสอ้างอิง
  • Pre-condition - เงื่อนไขก่อนทดสอบ
  • Steps - ขั้นตอนทีละขั้น
  • Test Data - ข้อมูลที่ใช้ทดสอบ
  • Expected Result - ผลลัพธ์ที่คาดหวัง

ตัวอย่าง Test Case

Field Value
IDTC_LOGIN_001
TitleLogin with valid credentials
Pre-conditionUser exists in system, on login page
Steps1. Enter username "john"<br>2. Enter password "password123"<br>3. Click Login button
ExpectedRedirect to dashboard, welcome message shown
