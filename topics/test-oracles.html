🔮 Test Oracle คืออะไร?

Test Oracle คือ แหล่งข้อมูลอ้างอิง ที่ใช้ในการตัดสินว่าผลลัพธ์จากการทดสอบ (Actual Result) ถูกต้องหรือไม่ เมื่อเทียบกับ Expected Result

💡 นิยามอย่างง่าย

Test Oracle = "เกณฑ์ตัดสิน" ว่าซอฟต์แวร์ทำงานถูกต้องหรือไม่

🎯 ทำไม Test Oracle สำคัญ?

  • ✅ กำหนด Expected Result ได้ชัดเจน
  • ✅ ตรวจสอบความถูกต้องของผลลัพธ์อัตโนมัติ
  • ✅ ช่วยใน Regression Testing

📋 ประเภทของ Test Oracles

📄

Specified Oracle

ใช้ specification/document เป็นเกณฑ์

เช่น: SRS, User Stories
💾

Derived Oracle

คำนวณ expected result จาก input

เช่น: 2+3=5 (คำนวณได้)
🔄

Implicit Oracle

ความรู้ทั่วไปที่ไม่ต้องอธิบาย

เช่น: App ไม่ควร crash
📊

Statistical Oracle

ใช้สถิติหรือ heuristic

เช่น: Response time < 2 วินาที

👤

Human Oracle

ผู้เชี่ยวชาญตัดสินใจ

เช่น: UX ดีไหม? (subjective)
🤖

Automated Oracle

ใช้เครื่องมือตรวจสอบ

เช่น: Assert equals, Screenshot compare

💡 ตัวอย่าง Test Oracles ในการทดสอบจริง

ตัวอย่าง 1: Calculator

# Test: add(2, 3)
# Oracle: Mathematical operation
# Expected: 5
assert calculator.add(2, 3) == 5  # ✓ Derived Oracle

ตัวอย่าง 2: Login System

# Test: Login with valid credentials
# Oracle: Requirements document
# Expected: Redirect to dashboard
# Actual: dashboard page loads
# Result: PASS ✓ (Specified Oracle)

ตัวอย่าง 3: Web Page Load

# Test: Page load time
# Oracle: Performance requirement (SLA)
# Expected: < 2 seconds
# Actual: 1.5 seconds
# Result: PASS ✓ (Statistical Oracle)

ตัวอย่าง 4: Visual Testing

# Test: UI appearance
# Oracle: Baseline screenshot
# Tool: Percy/Applitools compares images
# Result: PASS/FAIL based on diff % (Automated Oracle)
