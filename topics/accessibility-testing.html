♿ อะไรคือ Accessibility Testing?

Accessibility (a11y) คือการออกแบบและทดสอบให้ผู้พิการสามารถใช้งานเว็บไซต์/แอพได้ เช่น:

  • 👁️ ผู้พิการทางสายตา (ใช้ Screen Readers)
  • 👂 ผู้พิการทางการได้ยิน
  • ⌨️ ผู้ที่ใช้ Keyboard navigation
  • 🎨 ผู้ที่มองเห็นสีได้ยาก (Color blindness)

💡 ทำไมต้องสนใจ Accessibility?

  • 🌍 15% ของประชากรโลกมีความพิการ
  • ⚖️ กฎหมายบังคับในหลายประเทศ
  • 📈 ช่วย SEO และ User Experience ทั่วไป
  • 🏢 สร้าง Brand reputation

📋 WCAG (Web Content Accessibility Guidelines)

4 หลักการหลัก (POUR)

P

Perceivable

รับรู้ได้ - ข้อมูลต้องนำเสนอในรูปแบบที่ผู้ใช้สามารถรับรู้ได้

O

Operable

ใช้งานได้ - ส่วนติดต่อต้องใช้งานได้ด้วยวิธีต่างๆ

U

Understandable

เข้าใจได้ - ข้อมูลและการใช้งานต้องเข้าใจง่าย

R

Robust

ทนทาน - ต้องใช้งานได้กับเทคโนโลยี Assistive

ระดับความสอดคล้อง (Compliance Levels)

  • Level A - พื้นฐานที่สุด ต้องทำ
  • Level AA - มาตรฐานทั่วไปที่แนะนำ
  • Level AAA - สูงสุด ไม่บังคับทุกกรณี

✅ Accessibility Checklist

หัวข้อการทดสอบ
🔍 Alt Text รูปภาพต้องมี alt attribute ที่อธิบายเนื้อหา
🎨 Color Contrast Contrast ratio ต้อง ≥ 4.5:1 (AA) หรือ ≥ 7:1 (AAA)
⌨️ Keyboard Navigation ใช้ Tab, Enter, Space, Arrow keys ได้ครบ
🔤 Form Labels ทุก input ต้องมี label ที่เชื่อมโยงถูกต้อง
🎯 Focus Indicators เห็นว่า element ไหนกำลัง focus อยู่
📱 Resize Text ข้อความต้องอ่านได้เมื่อ zoom 200%
🔊 Screen Reader ทดสอบด้วย NVDA, JAWS หรือ VoiceOver
📝 Headings Structure Heading ต้องมีลำดับตรงกับเนื้อหา (h1→h2→h3)
