🔌 อะไรคือ API Testing?

API (Application Programming Interface) คือช่องทางที่แอพพลิเคชันต่างๆ สื่อสารกัน API Testing คือการทดสอบว่า API ทำงานถูกต้อง ตอบสนองตาม expected behavior

Client (Web/Mobile)  ←→  API Server  ←→  Database
        ↓                    ↓
   Frontend           Backend (ที่เราทดสอบ)

✨ ทำไมต้องทดสอบ API?

  • ⚡ เร็วกว่า UI Testing (ไม่ต้องรอหน้าเว็บโหลด)
  • 🔍 ตรวจสอบ Business Logic โดยตรง
  • 🛡️ ตรวจสอบ Security ที่ Data Level
  • 🔄 เหมาะสำหรับ CI/CD pipeline

POSTCreateสร้างข้อมูลใหม่POST /users

PATCHUpdateแก้ไขข้อมูลบางส่วนPATCH /users/123
DELETEDeleteลบข้อมูลDELETE /users/123

HTTP Status Codes

🟢 2xx Success

  • 200 OK - สำเร็จ
  • 201 Created - สร้างสำเร็จ
  • 204 No Content - สำเร็จ ไม่มีข้อมูลส่งกลับ

🟡 4xx Client Error

  • 400 Bad Request - Request ผิดรูปแบบ
  • 401 Unauthorized - ไม่ได้ login
  • 403 Forbidden - ไม่มีสิทธิ์เข้าถึง
  • 404 Not Found - ไม่พบ resource
  • 422 Unprocessable - Validation error

🔴 5xx Server Error

  • 500 Internal Error - Server มีปัญหา
  • 502 Bad Gateway - Gateway ผิดพลาด
  • 503 Service Unavailable - Server ไม่พร้อม

🚀 Postman - เครื่องมือทดสอบ API

การสร้าง Request พื้นฐาน

1. เปิด Postman
2. คลิก "+" สร้าง New Request
3. เลือก HTTP Method (GET, POST, etc.)
4. ใส่ URL: https://api.example.com/users
5. คลิก Send
6. ดู Response ด้านล่าง:
   - Status Code (200, 201, etc.)
   - Response Body (JSON)
   - Response Time

📤 POST Request with JSON Body

POST https://api.example.com/users

Headers:
Content-Type: application/json
Authorization: Bearer {token}

Body (raw - JSON):
{
    "name": "John Doe",
    "email": "john@example.com",
    "role": "user"
}

🔐 Postman Tests (Assertions)

// ไปที่ Tab "Tests" ใน Postman

// 1. ตรวจสอบ Status Code
pm.test("Status code is 200", function () {
    pm.response.to.have.status(200);
});

// 2. ตรวจสอบ Response Time
pm.test("Response time is less than 500ms", function () {
    pm.expect(pm.response.responseTime).to.be.below(500);
});

// 3. ตรวจสอบ JSON Schema
pm.test("Response has required fields", function () {
    var jsonData = pm.response.json();
    pm.expect(jsonData).to.have.property('id');
    pm.expect(jsonData).to.have.property('name');
    pm.expect(jsonData).to.have.property('email');
});

// 4. ตรวจสอบค่าที่ตอบกลับ
pm.test("User name is correct", function () {
    var jsonData = pm.response.json();
    pm.expect(jsonData.name).to.eql("John Doe");
});

// 5. ตรวจสอบ Header
pm.test("Content-Type is present", function () {
    pm.response.to.have.header("Content-Type");
});

🎯 สิ่งที่ต้องตรวจสอบใน API Test

หัวข้อ รายละเอียด ตัวอย่าง
Status Code ตรวจสอบ HTTP status ที่ตอบกลับ 200, 201, 400, 404, 500
Response Time เวลาตอบสนองต้องรวดเร็ว < 500ms, < 2s
Response Body ตรวจสอบ JSON structure และค่า มี field ครบ, ค่าถูกต้อง
Headers ตรวจสอบ Content-Type และ headers อื่นๆ application/json
Schema Validation ตรวจสอบว่าข้อมูลตรงกับ schema ที่กำหนด type, required fields
Error Handling ตรวจสอบกรณี error invalid input, not found

Test Case 1: GET Products List

// Test Case: Get All Products
// Method: GET
// URL: {{base_url}}/api/products

// 1. ตรวจสอบ Status Code
pm.test("Status code is 200", function () {
    pm.response.to.have.status(200);
});

// 2. ตรวจสอบ Response Time
pm.test("Response time is less than 500ms", function () {
    pm.expect(pm.response.responseTime).to.be.below(500);
});

// 3. ตรวจสอบ Content-Type
pm.test("Content-Type is application/json", function () {
    pm.response.to.have.header("Content-Type");
    pm.expect(pm.response.headers.get("Content-Type")).to.include("application/json");
});

// 4. ตรวจสอบ Response Body เป็น Array
pm.test("Response is an array", function () {
    const jsonData = pm.response.json();
    pm.expect(jsonData).to.be.an('array');
});

// 5. ตรวจสอบว่ามีสินค้าอย่างน้อย 1 รายการ
pm.test("Products array is not empty", function () {
    const jsonData = pm.response.json();
    pm.expect(jsonData.length).to.be.above(0);
});

// 6. ตรวจสอบ Schema ของสินค้าแต่ละรายการ
pm.test("Each product has required fields", function () {
    const jsonData = pm.response.json();
    jsonData.forEach(function(product) {
        pm.expect(product).to.have.property('id');
        pm.expect(product).to.have.property('name');
        pm.expect(product).to.have.property('price');
        pm.expect(product).to.have.property('stock');
        
        // ตรวจสอบ type
        pm.expect(product.id).to.be.a('number');
        pm.expect(product.name).to.be.a('string');
        pm.expect(product.price).to.be.a('number');
        pm.expect(product.stock).to.be.a('number');
        
        // ตรวจสอบค่า
        pm.expect(product.price).to.be.above(0);
        pm.expect(product.stock).to.be.at.least(0);
    });
});

// 7. เก็บ product ID ตัวแรกไว้ใช้ test ต่อไป
const firstProduct = pm.response.json()[0];
pm.collectionVariables.set("product_id", firstProduct.id);
pm.collectionVariables.set("product_price", firstProduct.price);

🔧 Pre-request Script (ตัวอย่าง)

// Pre-request Script - ทำก่อนส่ง request
// ใช้สำหรับเตรียมข้อมูล เช่น สร้าง timestamp, random data

// 1. สร้าง Unique Email
const timestamp = new Date().getTime();
const uniqueEmail = `test${timestamp}@example.com`;
pm.collectionVariables.set("unique_email", uniqueEmail);

// 2. สร้าง Random Quantity (1-5)
const randomQty = Math.floor(Math.random() * 5) + 1;
pm.collectionVariables.set("random_quantity", randomQty);

// 3. ตั้งค่า Header ถ้าต้องการ
pm.request.headers.add({
    key: 'X-Request-ID',
    value: pm.variables.replaceIn('{{$guid}}')
});

// 4. Console Log ดูค่า
console.log("Generated Email:", uniqueEmail);
console.log("Random Quantity:", randomQty);

ตัวอย่าง: Test API with Python

import requests
import pytest

BASE_URL = "https://api.example.com"

class TestUsersAPI:
    
    def test_get_users_list(self):
        """ทดสอบดึงรายชื่อ users"""
        response = requests.get(f"{BASE_URL}/users")
        
        # Assertions
        assert response.status_code == 200
        assert response.headers["Content-Type"] == "application/json"
        
        data = response.json()
        assert isinstance(data, list)
        assert len(data) > 0
    
    def test_create_user(self):
        """ทดสอบสร้าง user ใหม่"""
        payload = {
            "name": "Test User",
            "email": "test@example.com"
        }
        
        response = requests.post(
            f"{BASE_URL}/users",
            json=payload
        )
        
        assert response.status_code == 201
        
        data = response.json()
        assert data["name"] == payload["name"]
        assert data["email"] == payload["email"]
        assert "id" in data  # Server สร้าง id ให้
        
        # เก็บ id ไว้ใช้ test ต่อไป
        self.created_user_id = data["id"]
    
    def test_get_user_by_id(self):
        """ทดสอบดึง user ตาม id"""
        user_id = 1  # หรือใช้ self.created_user_id
        
        response = requests.get(f"{BASE_URL}/users/{user_id}")
        
        assert response.status_code == 200
        
        data = response.json()
        assert data["id"] == user_id
    
    def test_user_not_found(self):
        """ทดสอบกรณี user ไม่มีอยู่"""
        response = requests.get(f"{BASE_URL}/users/999999")
        
        assert response.status_code == 404
    
    def test_update_user(self):
        """ทดสอบแก้ไข user"""
        user_id = 1
        payload = {
            "name": "Updated Name"
        }
        
        response = requests.patch(
            f"{BASE_URL}/users/{user_id}",
            json=payload
        )
        
        assert response.status_code == 200
        assert response.json()["name"] == "Updated Name"
    
    def test_delete_user(self):
        """ทดสอบลบ user"""
        user_id = 1
        
        response = requests.delete(f"{BASE_URL}/users/{user_id}")
        
        assert response.status_code == 204  # หรือ 200
        
        # ตรวจสอบว่าลบจริง
        get_response = requests.get(f"{BASE_URL}/users/{user_id}")
        assert get_response.status_code == 404

✅ JSON Schema Validation

ตรวจสอบว่า Response มี structure ที่ถูกต้อง:

import jsonschema
from jsonschema import validate

# กำหนด Schema ที่ต้องการ
user_schema = {
    "type": "object",
    "properties": {
        "id": {"type": "integer"},
        "name": {"type": "string", "minLength": 1},
        "email": {"type": "string", "format": "email"},
        "role": {"type": "string", "enum": ["admin", "user", "guest"]},
        "active": {"type": "boolean"}
    },
    "required": ["id", "name", "email"]
}

# Test
def test_user_response_schema():
    response = requests.get(f"{BASE_URL}/users/1")
    data = response.json()
    
    # Validate schema
    validate(instance=data, schema=user_schema)
    
    # ตรวจสอบ type ด้วยตนเอง
    assert isinstance(data["id"], int)
    assert isinstance(data["name"], str)
    assert "@" in data["email"]  # Basic email check
