📚 Version Control คืออะไร?

Version Control System (VCS) คือระบบที่ใช้บันทึกการเปลี่ยนแปลงของไฟล์ เพื่อให้สามารถย้อนกลับไปดูหรือกู้คืนเวอร์ชันเก่าได้

💡 ทำไม QA ต้องรู้ Git?

  • ดูว่า bug เกิดจาก commit ไหน
  • ดึง code มา deploy/test เองได้
  • รีวิว code ก่อน merge
  • ทำงานร่วมกับ dev ได้ดีขึ้น

ประเภทของ Version Control

  • Local VCS - บันทึกบนเครื่องตัวเอง (เช่น RCS)
  • Centralized VCS - มี server กลาง (เช่น SVN, CVS)
  • Distributed VCS - ทุกคนมี repo ของตัวเอง (เช่น Git, Mercurial)

🌿 Git พื้นฐาน

Git เป็น Distributed Version Control ที่นิยมมากที่สุด พัฒนาโดย Linus Torvalds (ผู้สร้าง Linux)

🔧 Git Architecture

Working Directory
        ↓ (add)
   Staging Area
        ↓ (commit)
    Local Repo
        ↓ (push)
   Remote Repo
        (GitHub/GitLab)

📁 Git States

  • Working Directory - ไฟล์ที่กำลังแก้ไข
  • Staging Area (Index) - ไฟล์ที่เตรียม commit
  • Local Repository - ประวัติการ commit ในเครื่อง
  • Remote Repository - repository บน server (GitHub/GitLab)

💻 คำสั่ง Git ที่ใช้บ่อย

🚀 Setup & Basic Commands

# ตั้งค่า Git
git config --global user.name "Your Name"
git config --global user.email "your@email.com"

# สร้าง repo ใหม่
git init

# Clone repo จาก remote
git clone https://github.com/user/repo.git

# ดูสถานะ
git status

# เพิ่มไฟล์เข้า staging
git add filename.txt
git add .  # เพิ่มทั้งหมด

# Commit การเปลี่ยนแปลง
git commit -m "Add new feature"

# Push ขึ้น remote
git push origin main

# Pull การเปลี่ยนแปลงล่าสุด
git pull origin main

🌿 Branching

# ดู branch ทั้งหมด
git branch

# สร้าง branch ใหม่
git branch feature/login

# เปลี่ยน branch
git checkout feature/login
# หรือ
git switch feature/login

# สร้าง & เปลี่ยนในครั้งเดียว
git checkout -b feature/signup

# Merge branch
git checkout main
git merge feature/login

# ลบ branch
git branch -d feature/login  # ลบหลัง merge แล้ว
git branch -D feature/login  # บังคับลบ

🔍 Inspection Commands

# ดูประวัติ commit
git log
git log --oneline  # แบบสั้น
git log --graph    # แบบกราฟ

# ดู diff (ความแตกต่าง)
git diff
git diff --staged  # ดู staged changes

# ดูข้อมูล remote
git remote -v

# ดู commit ที่ทำกับไฟล์
git blame filename.txt

↩️ Undo Changes

# ยกเลิกการแก้ไขใน working directory
git checkout -- filename.txt

# ยกเลิกการ add (unstage)
git reset HEAD filename.txt

# แก้ไข commit ล่าสุด
git commit --amend

# ย้อนกลับไป commit ก่อนหน้า
git revert HEAD

🧪 Git สำหรับ QA Engineer

🐛 การหา Commit ที่ทำให้เกิด Bug (Git Bisect)

# เริ่ม bisect
git bisect start

# บอกว่า commit ปัจจุบันมี bug
git bisect bad

# บอกว่า commit เก่านี้ไม่มี bug
git bisect good v1.0

# Git จะ checkout commit ตรงกลางให้ทดสอบ
# ทดสอบแล้วบอกว่า good หรือ bad
git bisect good  # หรือ git bisect bad

# ทำซ้ำจนเจอ commit ที่ทำให้เกิด bug
git bisect reset  # ออกจาก bisect mode

📋 Git Workflow สำหรับ QA

  1. Clone repo — git clone https://...
  2. Pull ล่าสุด — git pull
  3. สร้าง branch สำหรับ test — git checkout -b test/feature-x
  4. เขียน/แก้ไข test scripts
  5. Commit การเปลี่ยนแปลง — git add . && git commit -m "Add tests"
  6. Push ขึ้น remote — git push origin test/feature-x
  7. สร้าง Pull Request
