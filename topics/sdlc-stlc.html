SDLC (Software Development Life Cycle)

กระบวนการพัฒนาซอฟต์แวร์ตั้งแต่เริ่มต้นจนถึงส่งมอบ ประกอบด้วย 6 ขั้นตอนหลัก:

1

Requirements Analysis

รวบรวมและวิเคราะห์ความต้องการจากลูกค้า

↓
2

Design

ออกแบบระบบและสถาปัตยกรรม

↓
3

Development

เขียนโค้ดตามที่ออกแบบ

↓
4

Testing

ทดสอบหาและแก้ไข Bugs

↓
5

Deployment

นำระบบขึ้น Production

↓
6

Maintenance

ดูแลและปรับปรุงระบบ

STLC (Software Testing Life Cycle)

กระบวนการทดสอบซอฟต์แวร์ที่สอดคล้องกับ SDLC แต่เน้นการทดสอบโดยเฉพาะ:

ขั้นตอน กิจกรรม Deliverables
1. Requirement Analysis วิเคราะห์ความต้องการที่ต้องทดสอบ RTM, Testable requirements
2. Test Planning วางแผน testing strategy, resources Test Plan, Estimation
3. Test Case Development เขียน test cases, scripts Test Cases, Test Data
4. Test Environment Setup เตรียม environment, tools Ready environment
5. Test Execution Run tests, report bugs Test Reports, Defect Reports
6. Test Closure สรุปผล, lessons learned Test Summary Report

SDLC Models

💧

Waterfall

ทำทีละขั้นตอน ไม่ย้อนกลับ เหมาะกับ requirements ที่ชัดเจน

🔄

Agile/Scrum

พัฒนาเป็น Sprint สั้นๆ ปรับเปลี่ยนได้เร็ว

🔁

Iterative

พัฒนาทีละรอบ ปรับปรุงไปเรื่อยๆ

V

V-Model

ทดสอบแต่ละ phase พร้อมกับการพัฒนา

บทบาท QA ในแต่ละ Phase

📋

Requirements

Review requirements, identify testable items, ask questions

📝

Design

Create test plan, identify test scenarios, prepare test data

🔍

Testing

Execute tests, report bugs, verify fixes, regression testing
