📈 ประเภท Performance Testing

⚡

Load Testing

ทดสอบด้วยโหลดปกติ (expected load)

  • จำลองผู้ใช้ตามจริง
  • ตรวจสอบ Response Time
  • Throughput ที่รองรับได้
🔥

Stress Testing

ทดสอบด้วยโหลดเกิน (beyond capacity)

  • หา Breaking Point
  • ตรวจสอบการ Recovery
  • พฤติกรรมตอน Crash
📈

Spike Testing

เพิ่มโหลดกะทันหัน (sudden spike)

  • Flash Sale scenario
  • Viral content
  • System resilience

🕐

Endurance Testing

ทดสอบนานๆ (soak test)

  • Memory leaks
  • Connection pool exhaustion
  • 8-24 ชั่วโมง+

🎯 Key Performance Metrics

Metric คำอธิบาย เป้าหมายดี
Response Time เวลาตอบสนองต่อ request < 2 seconds
Throughput จำนวน request/transaction ต่อวินาที สูงตาม SLA
Concurrent Users จำนวนผู้ใช้พร้อมกัน ตามที่ออกแบบ
Error Rate % ของ request ที่ผิดพลาด < 0.1%
CPU/Memory Usage การใช้ทรัพยากร server < 80%
Apdex Score ดัชนีความพึงพอใจผู้ใช้ (0-1) > 0.85

📐 Apdex Score Formula

(Satisfied Users + Tolerating Users/2) / Total Users

  • Satisfied: Response < T (threshold)
  • Tolerating: T ≤ Response < 4T
  • Frustrated: Response ≥ 4T

🔧 Apache JMeter

JMeter เป็น Open Source Load Testing Tool ที่นิยมใช้มากที่สุด

Basic JMeter Test Plan

Test Plan
├── Thread Group (Users)
│   ├── Number of Threads: 100
│   ├── Ramp-up: 60 seconds
│   └── Loop Count: 10
│
├── HTTP Request
│   ├── Protocol: https
│   ├── Server: api.example.com
│   ├── Method: GET
│   └── Path: /users
│
├── Listeners (Results)
│   ├── View Results Tree
│   ├── Summary Report
│   └── Graph Results

ขั้นตอนการใช้งาน

  1. เปิด JMeter → File → Templates → Building a Web Test Plan
  2. ตั้งค่า Thread Group (จำนวน user, ramp-up, duration)
  3. เพิ่ม HTTP Request Sampler
  4. เพิ่ม Listener ดูผลลัพธ์
  5. กด Run (Ctrl+R)
  6. วิเคราะห์ผลลัพธ์

⚡ k6 - Modern Load Testing (แนะนำ!)

// loadtest.js
import http from 'k6/http';
import { check, sleep } from 'k6';

// Test Configuration
export const options = {
  stages: [
    { duration: '2m', target: 100 },   // Ramp up
    { duration: '5m', target: 100 },   // Steady state
    { duration: '2m', target: 200 },   // Ramp up more
    { duration: '5m', target: 200 },   // Steady state
    { duration: '2m', target: 0 },     // Ramp down
  ],
  thresholds: {
    http_req_duration: ['p(95)<500'], // 95% ต้องเร็วกว่า 500ms
    http_req_failed: ['rate<0.1'],     // Error rate < 0.1%
  },
};

export default function () {
  const response = http.get('https://api.example.com/users');
  
  check(response, {
    'status is 200': (r) => r.status === 200,
    'response time < 500ms': (r) => r.timings.duration < 500,
  });
  
  sleep(1);
}

▶️ รัน k6

# รันทดสอบ
k6 run loadtest.js

# รันบน Cloud
k6 cloud loadtest.js

# รันด้วย Docker
docker run -i loadimpact/k6 run --vus 10 --duration 30s -< loadtest.js
