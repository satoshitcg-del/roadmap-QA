อินเทอร์เน็ตคืออะไร?

อินเทอร์เน็ต (Internet) คือเครือข่ายคอมพิวเตอร์ระดับโลกที่เชื่อมต่อกัน ผ่านชุดของโปรโตคอลมาตรฐานที่เรียกว่า TCP/IP ทำให้อุปกรณ์ต่างๆ สามารถสื่อสารและแลกเปลี่ยนข้อมูลกันได้

💡
สำหรับ QA: การเข้าใจอินเทอร์เน็ตช่วยให้คุณเข้าใจว่าทำไมแอปพลิเคชันถึงทำงานช้า หรือทำไมบางฟีเจอร์ถึงใช้งานไม่ได้ในบางภูมิภาค

องค์ประกอบหลักของอินเทอร์เน็ต

🔌

Infrastructure

  • สายใยแก้วนำแสง (Fiber Optic)
  • สายเคเบิล (Copper Cables)
  • ดาวเทียม (Satellites)
  • เสาสัญญาณ (Cell Towers)
📡

Protocols

  • TCP/IP - โปรโตคอลหลัก
  • HTTP/HTTPS - เว็บ
  • FTP - ไฟล์
  • SMTP - อีเมล
🖥️

Devices

  • Routers
  • Switches
  • Servers
  • End Devices (PC, Mobile)

IP Address & DNS

IP Address คืออะไร?

IP Address (Internet Protocol Address) คือที่อยู่เฉพาะที่กำหนดให้กับอุปกรณ์แต่ละเครื่องในเครือข่าย เปรียบเสมือน "บ้านเลขที่" บนอินเทอร์เน็ต

IPv4 IPv6
32-bit address 128-bit address
192.168.1.1 2001:0db8:85a3::8a2e:0370:7334
~4.3 billion addresses ~340 undecillion addresses
กำลังหมด อนาคต

DNS (Domain Name System)

DNS ทำหน้าที่เหมือน "สมุดโทรศัพท์" ของอินเทอร์เน็ต แปลงชื่อโดเมนที่มนุษย์อ่านได้ (เช่น google.com) เป็น IP Address ที่เครื่องคอมพิวเตอร์เข้าใจ

ตัวอย่างการทำงาน DNS 
# ตรวจสอบ DNS Record
$ nslookup google.com
Server:  8.8.8.8
Address: 8.8.8.8#53

Non-authoritative answer:
Name:   google.com
Address: 142.250.80.46

# ตรวจสอบผ่าน dig
$ dig google.com A +short
142.250.80.46

Client-Server Model

โมเดล Client-Server เป็นรูปแบบการสื่อสารหลักบนอินเทอร์เน็ต แบ่งบทบาทออกเป็นสองฝ่าย:

💻

Client

  • เบราว์เซอร์ (Chrome, Firefox)
  • Mobile Apps
  • ส่ง Request
  • รอรับ Response
  • แสดงผลให้ผู้ใช้
🖥️

Server

  • Web Server (Apache, Nginx)
  • Application Server
  • รอรับ Request
  • ประมวลผล
  • ส่ง Response กลับ
┌─────────────┐         Request          ┌─────────────┐
│             │ ────────────────────────>│             │
│   Client    │                           │   Server    │
│  (Browser)  │ <──────────────────────── │  (Backend)  │
│             │         Response          │             │
└─────────────┘                           └─────────────┘

มุมมอง QA: ทำไมต้องรู้?

🐌

ทำไมเว็บช้า?

เข้าใจว่าปัญหาอยู่ที่ DNS lookup, TCP connection, หรือ Server processing

❌

ไม่สามารถเข้าถึงได้

แยกแยะได้ว่าเป็น DNS error, Connection timeout, หรือ Server down

🌍

Geo-location Testing

เข้าใจว่า CDN กระจายเนื้อหาอย่างไร และ IP geolocation ทำงานอย่างไร

✅ Checklist สำหรับ QA

  • รู้ว่า DNS คืออะไรและทำงานอย่างไร
  • เข้าใจความแตกต่าง IPv4 vs IPv6
  • รู้ว่า Client-Server สื่อสารกันอย่างไร
  • ใช้ nslookup/dig ตรวจสอบ DNS ได้
  • อ่าน Network tab ใน DevTools ได้
