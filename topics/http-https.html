HTTP คืออะไร?

HTTP (HyperText Transfer Protocol) เป็นโปรโตคอลที่ใช้สื่อสารระหว่าง Client (เบราว์เซอร์) กับ Server บนอินเทอร์เน็ต เป็นรากฐานของการทำงานทั้งหมดบนเว็บไซต์

💡
สำหรับ QA: ต้องเข้าใจ HTTP เพื่อตรวจสอบ API, วิเคราะห์ Network Traffic และ Debug ปัญหาเว็บแอป

โครงสร้าง HTTP Request/Response

HTTP Request Example 
GET /api/users HTTP/1.1
Host: example.com
User-Agent: Mozilla/5.0
Accept: application/json
Authorization: Bearer token123
HTTP Response Example 
HTTP/1.1 200 OK
Content-Type: application/json
Content-Length: 123
Date: Mon, 24 Feb 2026 08:00:00 GMT

{
  "users": [
    {"id": 1, "name": "John"},
    {"id": 2, "name": "Jane"}
  ]
}

HTTP Methods

Method คำอธิบาย ใช้เมื่อไหร่
GET ดึงข้อมูล อ่านข้อมูล, ไม่มีผลต่อ Server
POST สร้างข้อมูล สร้าง Resource ใหม่
PUT แก้ไขข้อมูล (ทั้งหมด) อัปเดต Resource แทนที่ทั้งหมด
PATCH แก้ไขข้อมูล (บางส่วน) อัปเดตเฉพาะฟิลด์ที่เปลี่ยน
DELETE ลบข้อมูล ลบ Resource

HTTP Status Codes

2xx

Success

  • 200 OK - สำเร็จ
  • 201 Created - สร้างสำเร็จ
  • 204 No Content - สำเร็จแต่ไม่มีข้อมูล
3xx

Redirection

  • 301 Moved - ย้ายถาวร
  • 302 Found - ย้ายชั่วคราว
  • 304 Not Modified - ใช้ Cache
4xx

Client Error

  • 400 Bad Request
  • 401 Unauthorized
  • 403 Forbidden
  • 404 Not Found
  • 422 Validation Error
5xx

Server Error

  • 500 Internal Error
  • 502 Bad Gateway
  • 503 Service Unavailable

HTTP Headers สำคัญ

Common Headers 
# Request Headers
Content-Type: application/json    # ประเภทข้อมูล
Authorization: Bearer token       # ยืนยันตัวตน
Accept: application/json          # รับข้อมูลแบบไหน
User-Agent: Chrome/120            # เบราว์เซอร์

# Response Headers
Content-Length: 1234              # ขนาดข้อมูล
Cache-Control: max-age=3600       # การ Cache
Set-Cookie: session=abc123        # ตั้งค่า Cookie
