🌊 Waterfall Model

Waterfall เป็นโมเดลแบบลำดับขั้น (Sequential) ต้องทำขั้นตอนก่อนให้เสร็จก่อนจะไปขั้นตอนถัดไป

Requirements
    ↓
   Design
    ↓
Implementation
    ↓
  Testing
    ↓
Deployment
    ↓
Maintenance

⚠️ ข้อเสียของ Waterfall

  • ❌ ไม่สามารถย้อนกลับไปแก้ไขขั้นตอนก่อนหน้าได้ง่าย
  • ❌ ไม่เห็นงานจริงจนกว่าจะถึงขั้นตอนสุดท้าย
  • ❌ ไม่เหมาะกับ requirement ที่เปลี่ยนบ่อย
  • ❌ ความเสี่ยงสูงถ้าเข้าใจ requirement ผิดตอนแรก

✅ เหมาะกับ

  • ✓ โปรเจกต์ที่ requirement ชัดเจนและไม่เปลี่ยน
  • ✓ ระบบที่มีความเสี่ยงต่ำ
  • ✓ โปรเจกต์ที่ต้องมี documentation ละเอียด

🔺 V-Model (Verification & Validation Model)

V-Model เป็นขยายของ Waterfall ที่เน้นความสัมพันธ์ระหว่าง Development phase และ Testing phase แต่ละระดับมีการทดสอบที่สอดคล้องกัน

       Verification                    Validation
       (Development)                   (Testing)
       
       ┌─────────────┐          ┌─────────────┐
       │ Requirements│          │     UAT     │
       │    Specs    │◀────────▶│   Testing   │
       └──────┬──────┘          └──────┬──────┘
              │                        │
       ┌──────▼──────┐          ┌──────▼──────┐
       │    System   │          │    System   │
       │    Design   │◀────────▶│    Testing  │
       └──────┬──────┘          └──────┬──────┘
              │                        │
       ┌──────▼──────┐          ┌──────▼──────┐
       │   Module    │          │  Integration│
       │    Design   │◀────────▶│   Testing   │
       └──────┬──────┘          └──────┬──────┘
              │                        │
       ┌──────▼──────┐          ┌──────▼──────┐
       │    Coding   │◀────────▶│   Unit Test │
       └─────────────┘          └─────────────┘
              
                    ╲    /
                      ╲  /
                        ▼
                  Implementation

📊 V-Model Phases

Phase (Left) Testing (Right) คำอธิบาย
Requirements UAT (Acceptance Testing) ตรวจสอบว่าระบบตรงตาม business needs
System Design System Testing ทดสอบระบบทั้งหมดตาม design specs
Module/Architectural Design Integration Testing ทดสอบการเชื่อมต่อระหว่าง modules
Coding/Implementation Unit Testing ทดสอบแต่ละ unit/function โดย developer

💡 หลักการสำคัญ

ใน V-Model แต่ละ development phase มี testing phase ที่สอดคล้องกัน — ทำให้ตรวจพบปัญหาได้เร็วกว่า Waterfall

⚖️ เปรียบเทียบ SDLC Models

Model ลักษณะ Testing เหมาะกับ
Waterfall Linear, sequential ทำทีเดียวตอนท้าย Requirement แน่นอน
V-Model Sequential with parallel testing ทดสอบคู่ขนานทุก phase ระบบ mission-critical
Agile/Scrum Iterative, incremental Continuous testing Requirement เปลี่ยนบ่อย
Spiral Risk-driven, iterative ทดสอบทุก cycle โปรเจกต์มีความเสี่ยงสูง

✅ Verification vs Validation

🔍 Verification

"Are we building the product right?"

  • ตรวจสอบว่าทำตาม spec ถูกต้อง
  • ไม่ต้องรัน code
  • Static testing (reviews, inspections)
  • ตรวจ documents, code

ตัวอย่าง: Code review, requirements review

🎯 Validation

"Are we building the right product?"

  • ตรวจสอบว่าตรงตาม user needs
  • ต้องรัน code
  • Dynamic testing (execute software)
  • ทดสอบ functionality

ตัวอย่าง: UAT, System testing

📝 สรุปความแตกต่าง

VerificationValidation
ตรวจ spec/designตรวจ product จริง
Static (ไม่รัน)Dynamic (รัน code)
Developer/QAEnd User/QA
ตรวจก่อน buildตรวจหลัง build
